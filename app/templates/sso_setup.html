Azure AD Setup Instructions

Step 1: Create Azure AD App Registration

  1. Go to Azure Portal > App registrations
  2. Click "New registration"
  3. Enter a name: "GAM Web Interface"
  4. Select "Accounts in this organizational directory only"
  5. Set Redirect URI to: https://guigam.snowskeleton.net/auth/callback
  6. Click "Register"

Step 2: Configure API Permissions

  1. In your app registration, go to "API permissions"
  2. Add Delegated permissions:
    • • Click "Add a permission" > "Microsoft Graph" > "Delegated permissions"
    • • User.Read - Read user profile
    • • GroupMember.Read.All - Read group memberships
  3. Add Application permissions:
    • • Click "Add a permission" > "Microsoft Graph" > "Application permissions"
    • • Group.Read.All - Read all groups (for role mapping)
  4. Click "Grant admin consent" for all permissions (requires admin privileges)

Step 3: Create Client Secret

  1. Go to "Certificates & secrets"
  2. Click "New client secret"
  3. Enter description: "GAM Web Interface Secret"
  4. Set expiration (recommend 12 months)
  5. Click "Add" and copy the secret value immediately

Important: The client secret value is only shown once. Copy it now!

Step 4: Copy Required Values

From your app registration's "Overview" page, copy:

  • Application (client) ID - This is your Client ID
  • Directory (tenant) ID - This is your Tenant ID
  • Client secret value - From step 3

Step 5: Find Group IDs for Role Mapping

  1. Go to Azure Portal > Groups
  2. Find the groups you want to map to GUIGAM roles
  3. Click on each group and copy the "Object Id" from the overview page
  4. Use these Object IDs in the Role Mappings section above
